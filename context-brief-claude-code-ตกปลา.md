# Context Brief — สั่ง Claude Code สร้างระบบตกปลา (12 หาง)

> **เอกสารนี้คืออะไร:** brief สำหรับส่งให้ Claude Code สร้างระบบตกปลาเข้าเกมที่มีอยู่
> **เป้าหมาย:** สร้างระบบ + placeholder ให้เล่นได้จริงก่อน แล้วค่อยเจนอาร์ตมาทับทีหลัง
>
> **ใช้คู่กับ 2 ไฟล์นี้ (อ่านก่อนเริ่ม):**
> - `ระบบตกปลา-12หาง.md` — สเปคระบบ/กฎ/เศรษฐกิจทั้งหมด (source of truth)
> - `prompt-pack-ตกปลา-12หาง.md` — รายการ asset ที่จะเจนทีหลัง (บอกว่าต้องมี asset อะไรบ้าง)

---

## 0. คำสั่งแรกสุดสำหรับ Claude Code (ทำก่อนเขียนโค้ดใดๆ)

**อย่าเพิ่งเขียนโค้ด — สำรวจก่อน แล้วรายงานกลับ:**

1. อ่านไฟล์ `ระบบตกปลา-12หาง.md` และ `prompt-pack-ตกปลา-12หาง.md` ให้ครบ
2. สำรวจ repo เดิม แล้วสรุปกลับมาว่าเจออะไร:
   - เกมเรนเดอร์ 3D บนเว็บด้วยอะไร (three.js / babylon / PlayCanvas / Unity WebGL?)
   - เลเยอร์ UI/2D ปัจจุบันเป็น DOM/React ลอยทับ หรืออยู่ใน canvas เดียวกับ 3D
   - backend/server เป็นอะไร มี realtime อยู่แล้วไหม (world chat วิ่งบนอะไร — websocket?)
   - ระบบร้านค้า/สกุลเงิน Jil และช่องคอสตูม (สี/หน้า/อาวุธ/หมวก) เก็บ state ที่ไหน อย่างไร
   - inventory ของผู้เล่นเก็บที่ไหน
3. **หยุดถามยืนยันก่อน** ถ้าเจอสิ่งที่ขัดกับ assumption ด้านล่าง หรือถ้าเรื่องสกุลเงิน/inventory ไม่ชัด
4. เสนอแผน implement ตามเฟส (ข้อ 6) ให้ยืนยันก่อนลงมือ

> **Assumption ที่ต้องเช็ก** (แก้ได้ถ้าจริงไม่ตรง): โมเดลมาจาก Unity GLTF export, เรนเดอร์บนเว็บ, world chat มีผู้เล่นหลายคนพร้อมกัน, มีระบบร้านขายคอสตูมด้วยสกุลเงิน Jil อยู่แล้ว

## 1. กฎเหล็ก — ห้ามละเมิด (คัดจากสเปคระบบ)

1. **ตกปลาห้ามผลิต Jil เด็ดขาด** — Jil เป็นรายได้จริงจากการขายคอสตูม
   ตกปลาให้ **"เกล็ด" (สกุลเงินแยกใหม่)** เท่านั้น
2. **ห้ามแตะระบบคอสตูม/ร้าน Jil เดิม** — สร้างระบบใหม่ที่อยู่ข้างๆ ไม่แก้ของเดิม
   (ยกเว้นจุดเชื่อมที่จำเป็น เช่น อ่าน player id / inventory — ต้องถามก่อน)
3. **ผลตกปลาตัดสินที่ server** — client แค่เล่นอนิเมชันตามผล ห้ามสุ่มฝั่ง client
4. **แยกเลเยอร์ 2D ออกจาก 3D** — โมดูลตกปลา 2D ต้อง self-contained
   เลเยอร์ 3D รู้แค่ "อยู่ใกล้จุดตกปลาไหม" + โชว์ prompt เท่านั้น
5. **ไม่พึ่งอนิเมชันตัวละคร 3D** — ตัวละครแค่ค้างท่าถือคันเบ็ด อารมณ์ทั้งหมดอยู่ที่ 2D

## 2. หลักการที่ทำให้ "placeholder ก่อน เจนทีหลัง" เวิร์ค ⭐

**ทุก asset ต้องอ้างอิงผ่าน manifest กลางไฟล์เดียว — ห้าม hardcode path รูปในโค้ดเกม**

สร้าง `assetManifest` (เช่น JSON/TS config) ที่ map key → asset:

```
// asset-manifest ตัวอย่างแนวคิด (ปรับตาม stack จริง)
{
  "fish.common.carp":      { "sprite": "PLACEHOLDER", "tier": "common",   "color": "#9e9e9e" },
  "fish.legendary.dragon": { "sprite": "PLACEHOLDER", "tier": "legendary", "color": "#ffd700" },
  "rod.power.wood":        { "sprite": "PLACEHOLDER", "color": "#8d6e63" },
  "fx.catch.boom":         { "sprite": "PLACEHOLDER", "text": "BOOM" },
  "currency.scale":        { "sprite": "PLACEHOLDER", "color": "#26c6da" }
}
```

- ช่วง placeholder: renderer เห็น `"sprite": "PLACEHOLDER"` → วาดสี่เหลี่ยม/วงกลมสีตาม `color` + ตัวหนังสือ key/ชื่อ
- ช่วงมีอาร์ตจริง: เปลี่ยนแค่ `"sprite"` ให้ชี้ไฟล์รูปที่เจนมา → **โค้ดเกมไม่ต้องแก้แม้แต่บรรทัดเดียว**
- key ใน manifest ต้องตรงกับรายการ asset ใน `prompt-pack-ตกปลา-12หาง.md`

> นี่คือ contract หลัก: ระบบทำงานด้วย key ไม่ใช่ด้วยรูป → สลับอาร์ตเมื่อไรก็ได้

## 3. โครงสร้างข้อมูล (ให้ Claude Code ออกแบบตามนี้)

**Tier ความหายาก** (สีต้องตรงกับกรอบคอสตูมในร้าน):
`common (เทา) → rare (น้ำเงิน) → epic (ม่วง) → legendary (ทอง)`

**นิยามปลา (data-driven):**
```
Fish: { id, nameTH, tier, basePriceScale, spawnPoints[], assetKey }
```

**จุดตกปลา = trigger zone แบบ data ล้วน** (ห้าม raycast เช็กน้ำ):
```
FishingSpot: { id, position, lootTable[], condition? }
// condition เช่น time=night → ปลาเรืองแสง (เฟสหลัง)
```

**loot table ต่อจุด** = น้ำหนักความน่าจะเป็นของปลาแต่ละตัว (server ใช้สุ่ม)

**สกุลเงินใหม่:** `เกล็ด (scale)` — แยก field จาก Jil คนละตัวเด็ดขาด

---

## 4. Flow การตกปลา (server-authoritative)

```
[3D] ผู้เล่นเข้าใกล้ FishingSpot → โชว์ prompt "กดเพื่อตกปลา"
        ↓ กดปุ่ม
[Client→Server] ขอเริ่มตกปลา (spotId, playerId)
        ↓
[Server] ตรวจสิทธิ์ (ปลดคันเบ็ดแล้ว?) → สุ่มผลจาก lootTable → คำนวณ %จับ/ติด-หลุด
        ↓ ส่งผลกลับ (ปลาอะไร/tier/ติดหรือหลุด/ได้เกล็ดเท่าไร)
[Client 2D] เล่นมินิเกม + อนิเมชันตามผล (BOOM/MISS ตาม server)
        ↓
[Client] จบ → อัปเดต inventory + เกล็ด (จาก state ที่ server ยืนยัน)
```

- **%จับที่โชว์บนจอ = ค่าที่ server ส่งมา** ไม่ใช่ client คำนวณเอง
- มินิเกมจับจังหวะ (ถ้ามี) เป็นแค่ presentation — ผลจริงมาจาก server

---

## 5. ระบบปลดล็อกคันเบ็ดด้วยการออนไลน์

- ออนไลน์สะสมครบ 2 ชม./วัน → รับคันเบ็ด (วันละ 1 ครั้ง) **หรือ** ชวนเพื่อน 5 คน
- **การนับเวลาออนไลน์ต้องอยู่ฝั่ง server** (client แก้เวลาเองไม่ได้)
- ⚠️ **decision ยังไม่ล็อก** (ถามผู้ใช้ก่อน implement ส่วนนี้):
  ปลดครั้งเดียวจบ (ด่านแรกเข้าเกม) **หรือ** ต้องออนไลน์ทุกวันถึงตกได้ (daily gate)

## 6. ลำดับเฟส — สร้างตามนี้ (placeholder ก่อนเสมอ)

### เฟส 1 — Core loop เล่นได้ (ทั้งหมดเป็น placeholder)
- [ ] asset manifest + placeholder renderer (สี่เหลี่ยมสี tier)
- [ ] ข้อมูลปลา + FishingSpot + loot table (เริ่มปลา 4-6 ตัว, จุดตกปลา 1 จุด)
- [ ] 3D: detect เข้าใกล้จุด + โชว์ prompt (ยังไม่ต้องสวย)
- [ ] Server: สุ่มผลจาก loot table + ส่งกลับ
- [ ] 2D: มินิเกมพื้นฐาน + โชว์ %จับ + ผล ติด/หลุด (placeholder เอฟเฟกต์เป็นตัวหนังสือ)
- [ ] เก็บปลาเข้า inventory + ได้เกล็ด (สกุลเงินแยกจาก Jil)
- [ ] ขายปลา → ได้เกล็ด (ราคาคงที่ตาม tier ก่อน ยังไม่ทำตลาดผันผวน)
- **Definition of done:** เข้าไปกดตกปลา ได้ปลา ขายได้เกล็ด ครบลูป ด้วย placeholder ล้วน

### เฟส 2 — รางวัล & ปลดล็อก
- [ ] ปลดของ exclusive จากความสำเร็จ (เส้นรางวัลที่ 2 จากสเปค — ทำก่อนสกุลเงินเต็ม)
- [ ] ระบบปลดคันเบ็ดด้วยเวลาออนไลน์ (นับฝั่ง server) — **หลังยืนยัน decision ข้อ 5**
- [ ] สายอัปคันเบ็ด (ซื้อด้วยเกล็ด, เพิ่มพลัง)

### เฟส 3 — เอาอาร์ตจริงมาทับ
- [ ] เจนปลา/คัน/UI ตาม `prompt-pack-ตกปลา-12หาง.md`
- [ ] อัปเดต `sprite` ใน manifest ให้ชี้ไฟล์จริง (ไม่แตะ logic)
- [ ] คอมิกเอฟเฟกต์จริง (BAM/POW/MISS/BOOM) แทน placeholder ตัวหนังสือ

### เฟส 4 — ของเติมตอนโต (ทำเมื่อคนเยอะ)
- [ ] ตู้ปลา/ของโชว์ + ของพรีเมียมสายตกปลาที่ขายด้วย Jil (หนุนยอด Jil)
- [ ] ตลาดราคาผันผวน (เปิดเมื่อ volume มากพอ)
- [ ] จุดตกปลาตามเวลา/สภาพแวดล้อม (กลางคืน/ฝน)

## 7. สิ่งที่ห้ามทำ (สรุปย้ำ)

- ❌ ห้ามให้ตกปลาออก Jil ไม่ว่ารูปแบบไหน
- ❌ ห้าม hardcode path รูปในโค้ดเกม (ต้องผ่าน manifest)
- ❌ ห้ามสุ่มผลตกปลาฝั่ง client
- ❌ ห้ามแก้ระบบคอสตูม/ร้าน Jil เดิมโดยไม่ถาม
- ❌ ห้ามผูกระบบตกปลากับ rig/animation ตัวละคร 3D
- ❌ ห้าม raycast เช็กน้ำ (ใช้ trigger zone แบบ data)
- ❌ ห้ามเจนอาร์ตก่อนกลไกนิ่ง (placeholder ก่อนเสมอ)

---

## 8. Definition of Done — เฟส 1 (เกณฑ์ส่งมอบ)

ระบบถือว่าผ่านเฟสแรกเมื่อ:
- เดินตัวละครเข้าใกล้จุดตกปลา → เห็น prompt
- กด → server สุ่ม → 2D เล่นผล → ได้/ไม่ได้ปลา
- ปลาเข้า inventory, เกล็ดเพิ่ม (Jil ไม่ขยับ)
- ขายปลาได้เกล็ดตาม tier
- **ทุกภาพเป็น placeholder** และสลับเป็นอาร์ตจริงได้โดยแก้แค่ manifest
- ไม่มีจุดไหนแตะโค้ดระบบคอสตูมเดิม

> เมื่อผ่านเฟส 1 = เอา placeholder ไปให้คนในคอมมูลองเล่นได้ทันที เช็กว่ากลไกสนุกก่อนลงแรงเจนอาร์ต
